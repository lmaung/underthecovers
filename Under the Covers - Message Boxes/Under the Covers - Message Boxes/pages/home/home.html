Message Boxes

The first item that developers normally need in developing or testing an app is a way to display information back to the user interface for various debug messages. Typically, we use messagebox.show or alert to prompt a display.

In the case of Windows 8, afore mentioned methods can no longer be used since Windows 8 expect the prompts to be handled seperately thus, freeing the UI or core app to continue on.

Normally, in the cases where UI code is different from a seperate portions of codes where another activity is taking place (in our case, message box), we have to create a seperate thread and display that accordingly.

In Windows 8, built in ASYNC and AWAIT will take care of creating, and monitoring seperate threads and processes.
